# PancreaticClassification_with_NN

# Pancreatic Classification with Neural Networks

## Описание
Ноутбук посвящён задаче бинарной классификации пациентов по риску заболеваний поджелудочной железы на основе клинических и биомаркерных признаков. В работе сравниваются базовая логистическая регрессия и несколько вариантов нейронных сетей.

## Данные
Используется датасет **Debernardi et al., 2020** с биомаркерами и клиническими признаками пациентов.

Исходные признаки включают:
- возраст;
- пол;
- уровень `plasma_CA19_9`;
- `creatinine`;
- `LYVE1`;
- `REG1B`;
- `TFF1`;
- `REG1A`.

Целевая переменная приводится к бинарному виду:
- `1` — пациент болен;
- `0` — пациент здоров.

Размер выборки после подготовки:
- train: **472** объектов;
- test: **118** объектов.

## Что сделано в ноутбуке

### 1. Первичный анализ данных
- загрузка и просмотр датасета;
- проверка пропусков;
- анализ распределения классов;
- просмотр категориальных признаков.

### 2. Предобработка
- бинаризация признака `sex`;
- логарифмирование числовых признаков через `np.log1p`;
- заполнение пропусков нулями;
- отбор признаков без target leakage;
- разделение данных на train/test в пропорции 80/20 со stratify;
- min-max нормализация признаков.

### 3. Базовая модель
Обучается **логистическая регрессия** как baseline.

Результат на тестовой выборке:
- accuracy: **0.78**;
- recall для класса болезни: **0.75**;
- precision для класса болезни: **0.91**;
- weighted F1-score: **0.79**.

### 4. Нейросетевые модели
В ноутбуке последовательно исследуются:
- однослойная нейросеть;
- двухслойная нейросеть;
- трёхслойная нейросеть;
- влияние `weight_decay`;
- влияние `BatchNorm1d`;
- сравнение функций активации: `ReLU`, `Tanh`, `Sigmoid`.

Для обучения используются:
- **PyTorch**;
- `BCEWithLogitsLoss`;
- оптимизаторы **SGD** и **Adam**;
- early stopping;
- мониторинг loss и accuracy.

## Основные результаты
- однослойная сеть показывает результат ниже более глубоких архитектур;
- двухслойная сеть даёт accuracy около **0.82**;
- трёхслойная сеть улучшает качество до **0.85**;
- добавление batch normalization повышает метрику примерно до **0.86–0.87**;
- лучшая функция активации в эксперименте — **ReLU**.

Сравнение активаций:
- ReLU: **0.864**;
- Tanh: **0.839**;
- Sigmoid: **0.797**.

## Вывод
В рамках этого ноутбука нейросетевые модели показали лучший результат, чем логистическая регрессия. Лучшее качество достигается у трёхслойной сети с batch normalization, регуляризацией и активацией ReLU.
